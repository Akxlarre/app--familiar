# Component Discovery

Antes de crear o modificar componentes de UI:

## 1. Buscar componentes existentes
- Consultar `src/app/shared/components/COMPONENTS.md`
- Verificar si existe un componente que cubra el caso de uso

## 2. Leer documentación
- Si existe el componente: leer su README.md, ARCHITECTURE.md o *.md en su carpeta
- Usar el componente existente en lugar de crear uno nuevo o duplicar lógica

## 3. Solo crear nuevo cuando
- No exista componente en COMPONENTS.md que cubra el caso
- PrimeNG no tenga equivalente adecuado
- Sea necesario un comportamiento muy especializado
